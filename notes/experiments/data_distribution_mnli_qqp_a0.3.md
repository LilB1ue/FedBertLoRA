# Per-client data distribution: MNLI / QQP × α=0.3

> 最後更新：2026-05-01
> 補完 SST-2 / QNLI 之外另兩個主要 GLUE task 的 per-client stats（30 clients, Dirichlet α=0.3, seed=42）。

## Columns

- `pid`: partition_id (0..29)
- `n_train`, `n_test`: 訓練/測試集大小（stratified 80/20 split）
- `train L0/L1[/L2]`、`test L0/L1[/L2]`: 各 label 的 sample 數
- `dominant` (MNLI) / `label_1 ratio` (QQP): 最大 label 占比 + 標籤類型

Badge：
- 🔴 ≥ 95%（極端 dominant）
- 🟣 ≤ 5% label_1（QQP only，極端反向）
- 🟡 ≥ 80%（重度偏向）
- ⚖ ≤ 40% max ratio（MNLI only，相對均衡）

## 觀察

1. **MNLI 是 3-class，extreme client 形態複雜**：每個 client 可能 dominant 在 L0/L1/L2 任一個，產生 3 個方向的 cluster 而非 binary 的兩極化。
2. **QQP 樣本數最大**（n_train > 290K），且 pid_11 + pid_17 兩個極端反向 client 都是大型（>26K）— 可能造成跟 QNLI α=0.5 pid_16 類似但更顯著的 structural singleton。
3. **MNLI 跟 QQP 的 max client 都比 SST-2/QNLI 大很多**（34K 跟 59K vs 7K 跟 10K）— 訓練成本明顯增加。
4. **pid_11 (QQP)** 是 label_0 = 99.99%，且 n = 59734 — 這是**史上最強的 inverse-marginal outlier 候選**，預期會 structural singleton 跟 pid_17 形成 binary 對立。
